# ユーザーテンプレート

このディレクトリでは、Spec Workflow のデフォルトテンプレートを上書きするカスタムテンプレートを作成できます。

## カスタムテンプレートの使用方法

1. **カスタムテンプレートファイルを作成**します。上書きしたいデフォルトテンプレートと全く同じ名前でこのディレクトリに作成してください：

   - `requirements-template.md` - 要件定義書テンプレートの上書き
   - `design-template.md` - 設計書テンプレートの上書き
   - `tasks-template.md` - タスク書テンプレートの上書き
   - `product-template.md` - プロダクト指針テンプレートの上書き
   - `tech-template.md` - 技術指針テンプレートの上書き
   - `structure-template.md` - 構造指針テンプレートの上書き

2. **テンプレート読み込みの優先順位**：
   - システムは最初にこの `user-templates/` ディレクトリをチェックします
   - ここで一致するテンプレートが見つかった場合、それが使用されます
   - それ以外の場合は、`templates/` のデフォルトテンプレートが使用されます

## カスタムテンプレートの例

カスタム要件定義書テンプレートを作成するには：

1. このディレクトリに `requirements-template.md` という名前のファイルを作成します
2. カスタム構造を追加します。例えば：

```markdown
# 要件定義書

## エグゼクティブサマリー

[カスタムセクション]

## ビジネス要件

[カスタム構造]

## 技術要件

[カスタムフィールド]

## カスタムセクション

[ワークフロー固有のセクションを追加]
```

## テンプレート変数

テンプレートには、ドキュメント作成時に置換されるプレースホルダーを含めることができます：

- `{{projectName}}` - プロジェクト名
- `{{featureName}}` - 仕様化する機能の名前
- `{{date}}` - 現在の日付
- `{{author}}` - ドキュメント作成者

## ベストプラクティス

1. **デフォルトから開始する**: 開始点として `../templates/` からデフォルトテンプレートをコピーします
2. **構造の一貫性を保つ**: ツールの互換性のために、同様のセクションヘッダーを維持します
3. **変更を文書化する**: セクションが追加・変更された理由をコメントで説明します
4. **バージョン管理**: カスタムテンプレートをバージョン管理で追跡します
5. **徹底的にテストする**: カスタムテンプレートが spec workflow ツールで動作することを確認します

## 注意事項

- カスタムテンプレートはプロジェクト固有であり、パッケージ配布には含まれません
- `templates/` ディレクトリには、各バージョンで更新されるデフォルトテンプレートが含まれています
- このディレクトリのカスタムテンプレートは、アップデート時も保持されます
- カスタムテンプレートにエラーがある場合、システムはデフォルトテンプレートにフォールバックします
